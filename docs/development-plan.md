# 开发计划

## 1. 总体策略

先跑通“能弹、能播、能导入、能同步显示”的主链路，再做复杂编辑和图片识谱。  
不要一开始就做完整乐谱编辑器，那会显著拖慢项目进度。

推荐开发顺序：

1. 演奏体验
2. 播放引擎
3. 曲谱内部模型
4. 输入与导入
5. 曲谱预览
6. 图片识谱

## 2. 阶段拆分

## Phase 0：项目初始化

目标：

- 初始化前端工程
- 搭建基础目录结构
- 接入 `PixiJS` 与 `Tone.js`
- 建立基础状态管理和类型定义

任务：

- 初始化 `Next.js + TypeScript`
- 配置 `eslint` / `prettier`
- 建立 `src/modules` 分层
- 定义 `ScoreDocument`、`NoteEvent` 等核心类型

验收标准：

- 项目可本地启动
- 页面可渲染空白钢琴容器
- 类型系统可通过检查

## Phase 1：88 键钢琴演奏 MVP

目标：

- 在网页中展示 88 键钢琴
- 支持鼠标与触摸点击发声
- 支持按键高亮和基础动画

任务：

- 生成 88 键布局数据
- 使用 `PixiJS` 绘制白键与黑键
- 接入 `Tone.js` 采样音源
- 建立 `triggerNote(midi, velocity)` 接口

验收标准：

- 88 键正确显示
- 点击任意键可发声
- 黑白键层级与命中区域正常
- 支持连续快速点击

## Phase 2：播放时间轴与自动演奏

目标：

- 让应用可以根据内部曲谱数据自动演奏
- 在播放时同步高亮钢琴键

任务：

- 构建 `NoteEvent[]` 调度器
- 接入 `Tone.Transport`
- 支持播放、暂停、停止、从指定位置开始
- 实现当前 tick 到 UI 状态的同步

验收标准：

- 给定一组 `NoteEvent[]` 可以稳定自动播放
- 和弦、连续音、休止都能按节奏执行
- 播放状态切换不会错位

## Phase 3：输入与导入

目标：

- 提供至少两种曲谱进入系统的方式

任务：

- 实现简易文本语法解析器
- 支持 `MIDI` 文件导入
- 将导入结果统一转换为 `ScoreDocument`
- 为解析错误提供明确提示

验收标准：

- 用户可以贴一段文本直接生成可播放曲谱
- 用户可以上传一个简单 `MIDI` 文件并播放
- 导入失败时能给出可理解的错误信息

## Phase 4：五线谱预览与校对

目标：

- 让用户不只听得到，还看得到谱

任务：

- 接入 `OpenSheetMusicDisplay`
- 支持播放时的当前小节或音符高亮
- 做一个简易“音符检查面板”
- 支持基础级别的单音修正

验收标准：

- 导入曲谱后可显示五线谱
- 播放进度能和谱面同步
- 用户能定位到某个问题小节

## Phase 5：图片/PDF 识谱导入

目标：

- 让用户上传网络乐谱图片后得到可播放结果

任务：

- 接入 OMR 服务
- 建立上传与任务状态流程
- 将 OMR 输出转为 `MusicXML`
- 对低置信度区域做标记

验收标准：

- 清晰印刷体乐谱可完成基础识别
- 用户可以看到哪些位置需要人工复核
- 修正后可重新播放

## Phase 6：体验打磨与长期能力

目标：

- 提升产品可用性和留存能力

任务：

- 作品保存
- 最近文件记录
- 速度控制和循环练习
- 外接 MIDI 键盘
- 移动端交互优化

验收标准：

- 用户可以重复使用自己的曲谱
- 常用练习流程更顺手

## 3. 推荐的近期执行清单

如果我们下一步就开始写代码，我建议按下面顺序推进：

1. 初始化 `Next.js` 工程
2. 做出 `PixiJS` 88 键静态布局
3. 接入 `Tone.js` 让点击可发声
4. 定义曲谱核心类型与示例数据
5. 用一组假数据跑通自动演奏
6. 再补文本输入和 `MIDI` 导入

## 4. 风险与应对

### 风险 1：曲谱格式过早做复杂

应对：

- 第一版先用内部简化模型
- 导入层再兼容 `MIDI` / `MusicXML`

### 风险 2：图片识谱准确率不稳定

应对：

- 只把它当“辅助导入”
- 一定保留人工校对链路

### 风险 3：钢琴 UI 和乐谱 UI 相互牵制

应对：

- 明确 `PixiJS` 只负责演奏舞台
- 乐谱显示走独立组件体系

## 5. 建议的里程碑定义

### 里程碑 A：可以弹

- 88 键可见
- 点键即发声

### 里程碑 B：可以播

- `NoteEvent[]` 自动演奏可用
- 键盘高亮同步

### 里程碑 C：可以导入

- 文本输入可用
- `MIDI` 导入可用

### 里程碑 D：可以看谱

- 曲谱预览可用
- 播放跟随可用

### 里程碑 E：可以识谱

- 图片/PDF 导入后可进入修正流程
