# 产品拆解

## 1. 产品目标

做一个网页端电子钢琴应用，核心能力包括：

- 提供接近普通电子琴体验的 88 键演奏界面
- 支持用户手动点击/触摸弹奏
- 支持导入或输入曲谱后自动演奏
- 在播放时同步展示按键、节奏和曲谱状态
- 逐步支持从图片或 PDF 乐谱中识别曲谱并导入

这个产品的第一阶段重点不是“做一个很完整的乐谱软件”，而是先做出一个“可以演奏、可以播放、可以导入、用户不费劲”的在线钢琴应用。

## 2. 核心用户场景

### 场景 A：自由弹奏

用户打开网页后，直接点击钢琴键或使用外接 MIDI 键盘演奏，页面实时发声并高亮按键。

### 场景 B：输入曲谱并自动演奏

用户通过文本、文件或简易编辑方式输入一段曲谱，系统解析后自动演奏，并在钢琴键和曲谱上同步显示。

### 场景 C：导入已有曲谱

用户上传 `MusicXML`、`MIDI`，或后续上传图片/PDF 乐谱，系统将其转换为统一内部格式，再播放和预览。

### 场景 D：识谱后校对

用户上传图片乐谱后，系统给出识别结果与疑似错误提示，用户修正后再播放。

## 3. 功能模块拆分

## 3.1 演奏舞台层

主要由 `PixiJS` 承担：

- 88 键键盘绘制
- 白键/黑键的布局、状态、高亮和按压动画
- 鼠标、触摸拖动、多指点击
- 自动演奏时的键位跟随
- 未来可扩展掉落音符或 piano roll 视图

## 3.2 音频播放层

主要由 `Tone.js` 承担：

- 钢琴采样加载
- 单音和和弦播放
- 节拍和时间轴调度
- 暂停、继续、跳转、速度调整
- 节拍器和循环播放

## 3.3 曲谱领域层

这是整个项目最关键的中间层，负责把不同来源的曲谱统一成一套内部数据：

- 外部标准格式：`MusicXML`
- 兼容导入格式：`MIDI`
- 内部播放格式：`NoteEvent[]`
- 内部编辑容器：`ScoreDocument`

## 3.4 输入与导入层

建议按难度和收益分级：

### 第一优先级

- 点击钢琴录入
- 简易文本输入
- `MusicXML` / `MIDI` 文件导入

### 第二优先级

- 五线谱预览后的逐音修正
- 拖动调整音高与时值

### 第三优先级

- 图片 / PDF 乐谱识别导入

## 3.5 曲谱展示与校对层

建议使用独立乐谱渲染方案，而不是让 `PixiJS` 硬做五线谱排版：

- 五线谱预览
- 当前小节高亮
- 当前音符高亮
- 小节、拍子、速度信息展示
- 识谱低置信度区域提示

## 3.6 后端与存储层

当功能进入可用阶段后再逐步补齐：

- 用户曲谱保存
- 最近使用记录
- 图片识谱任务处理
- 文件上传与转码
- 未来的分享与公开曲谱库

## 4. MVP 范围建议

## 4.1 第一版必须有

- 网页可打开的 88 键钢琴
- 点击按键发声
- 自动播放 `NoteEvent[]`
- 导入至少一种文件格式，推荐先支持 `MIDI`
- 一个简易的文本输入方式
- 播放、暂停、停止、调速
- 播放时钢琴键同步高亮

## 4.2 第一版可以暂时不做

- 完整五线谱可视化编辑器
- 高精度图片识谱
- 用户体系和账号系统
- 多人协作
- 手机端完整专业编辑体验

## 5. 输入方案建议

为了让用户“输入曲谱舒服”，建议不要只提供一种入口，而是同时做三层入口：

### 入口 1：最简单的文本输入

适合快速验证功能，可以先设计一种比 `MusicXML` 轻很多的简易语法，例如：

```txt
tempo=90
RH: C4 q, E4 q, G4 h
LH: C3 h, G2 h
```

这里的 `q` 表示四分音符，`h` 表示二分音符。

### 入口 2：文件导入

适合已有数字曲谱的用户：

- `MIDI`：最容易先支持，适合自动演奏
- `MusicXML`：更适合保留五线谱结构

### 入口 3：图片/PDF 导入

适合网上找来的乐谱，但必须接受“识别后需要校对”的现实：

- 上传图片/PDF
- 后端识谱转换
- 标出低置信度小节
- 用户修正
- 再进入播放流程

## 6. 用户体验原则

- 输入后立刻能听到结果，减少“写了但没反馈”的等待感
- 当前播放到哪里，要同时体现在键盘、时间轴和曲谱上
- 错误提示要定位到具体小节或具体音，而不是只说“解析失败”
- 手机端优先保证“能弹、能看、能播”，不要强求一次性编辑完整 88 键全视图
- 图片识谱必须允许人工校对，不能承诺一键正确

## 7. 关键风险

- 五线谱编辑本身复杂，不适合第一版就全做
- 图片识谱准确率受图片质量影响很大
- 多声部、踏板、连音线、装饰音会显著提升数据模型复杂度
- `PixiJS` 很适合钢琴和动画，但不适合直接充当复杂乐谱编辑器
